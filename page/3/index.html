JAVA

5）安全性
Java被设计能够防范以下问题：

  • 运行时堆栈溢出；
  • 自己的处理空间之外破坏内存；
  • 未经授权读写文件；有数字签名类；
  • 在程序运行时加以控制并制止恶意破坏，与ActiveX比起来更安全。

6）体系结构中立
Java编译器通过生成与特定的计算机体系结构无关的字节码指令，来生成一个体系结构中立的目标文件格式。
可以将使用最频繁的字节码序列翻译成机器码（即时编译）。

7）可移植性
Java的数据类型的大小和算法都有明确规定，字符串用标准的Unicode存储；
类库定义了可移植的接口；
用户界面工具箱虽然不好看但很实用，并且现在已经完全重写，更稳定。

8）解释型
移植了解释器的机器上都可以执行Java字节码，现在使用即时编译器将字节码翻译成机器码。

9）高性能
即适用性更强。字节码可以在运行时刻快速翻译成应用程序的特定CPU的机器码。
即时编译器可以监控经常执行哪些代码并优化代码以提高速度，或者何时内嵌何时消除内嵌，甚至撤销优化。

10）多线程
只要操作系统支持，Java中的线程就可以利用多个处理器；
它把多线程的实现交给了底层的操作系统或线程库来完成；
多线程编译的简单性让Java成为服务端开发语言。

11）动态性
库可以自由添加新方法和实例变量；
可以往正在运行的程序中添加代码；
可以在程序运行时获得对象的结构和行为等类型信息。

JAVA

1）简单性

简单性是与C++对比得出的特性。它没有C++的头文件、指针运算、结构、联合、操作符重载、虚基类等。
Java还能支持开发能够在小型机器上独立运行的软件。

2）面向对象
面向对象特性与C++相当，不同点在于多继承。Java使用的是简单的接口概念和元类模型。

3）网络技能
能够通过URL很方便地打开访问网络上的对象。

4）健壮性
与C++的不同在于，Java采用的指针模型可以消除重写内存和损坏数据的可能，不会出现内存分配错误，也不必防范内存泄漏。
Java编译器能够检测其他语言在运行时才能发现的问题。
