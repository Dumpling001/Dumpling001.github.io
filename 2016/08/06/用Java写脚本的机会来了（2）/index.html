用Java写脚本的机会来了（2）

发表于 2016-08-06   |  

一、脚本进行

周五，准备写脚本了。看哪些地方该替换属性。基本就是顺藤摸瓜找调用的方法的过程。原来配置文件不需要改，原来只是要对其中一个app发送指令类的方法里面的参数进行修改。

于是硬着头皮看主要的方法调用了哪些方法，一层一层地找。

开始老大说设备不校验参数，于是试着只替换已有的3个参数，调试运行，不行；

再问，又说是全量下发，也就是说所有属性都要传参数。于是又改，把所有参数都补上，再把方法里调用的json对象生成方法也全部更新成目标设备的参数，再试，不行；

原来是方法里增加了参数，但调用时没有带参数；修改，也不行；

干脆，不调用参数，改传固定参数。这下可好了，服务端查到的日志里，显示一大堆的杂乱无章的字符串，充满了整个屏幕。不过，在这一大堆字符串里，我发现10个参数里，前面的3个参数传上去了。

折腾了一下午，终于弄明白了，原来10个参数里，有4个参数是Json格式，而我传的是一个简单的字符串。格式不匹配，难怪呢。而这个Json格式又是这个设备所特有的，所以我用原来的方法生成json对象不行。老大说周一来指导下，那就等周一了。今天还是有进步的，起码有30%的参数传成功了，哈哈。

二、 饭局

我们客户公司的一个开发同事晋升了，晚上请吃饭，叫了我们团队的两个人，开始我并不想去，后来听说晋升就答应去了。我们俩提前下班，先随他先去定位置。聚餐的大多数人是客户自己公司的同事，他们来得晚，迟到了一个小时，吃完后聊些我听不懂的前期开发事项，呆着很无趣，到九点多才散，弄得晚上回家很晚。至少浪费了4个小时，第二天精神萎靡不振。希望下次能够礼貌而又坚决的拒绝这些饭局。

三、 重建博客

晚上又想把统计功能弄出来，试过百度统计、Google统计、腾讯统计，始终不能显示。倒腾了2小时，结果出了点问题。我干脆把本地整个文件夹删掉了。因为以前有过类似经验，删掉后，再从github上克隆下来，也可以恢复原状。不过这次可惨了，删掉后再克隆，然后生成部署，一切都烟消云散了：3篇博文、修改过的站点配置文件、主题配置文件全没了。

一招回到解放前。

于是又从头开始部署、修改。不过这次比以前顺利多了。这次不再去找那些教程了，直接上Hexo、Next网站上看指导文档，按说明操作，感觉比那些教程要清晰、全面多了。附上链接：

http://theme-next.iissnan.com/
https://hexo.io/zh-cn/docs/

统计显示仍然没解决，算了放弃吧。不过，把头像给显示出来了，还是有点小成就的。
