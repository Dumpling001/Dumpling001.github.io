用Excel和Python实现数据分析笔记之初体验

发表于 2016-10-31   |  

本文将回顾Excel实现数据分析，和Python实现数据分析的基本流程，并记录使用Python实现时遇到的两个坑。

Excel简单实现数据分析的基本流程

  1. 将train.csv文件导入Excel；
  2. 进入数据透视表界面；
  3. 用直觉猜测模式，例如计算性别幸存比例；
  4. 把Survived变量作为2个值标签，一个是总个数，一个是值的和，性别作为行标签，计算男女幸存比例，印证直觉；
  5. 预测模型为：性别为女即幸存；
  6. 打开test.csv,插入Survived列，用if逻辑表达式实现预测模型；
  7. 删除Survived和PassengerID以外的列，另存为文件genderbasedmodel.csv；
  8. 在Kaggle上提交该文件。

Excel实现较复杂数据分析的基本流程

  • 在上面的第4步中，行标签在性别以外再添加票价，等级，分别计算幸存者比例；
  • 将计算出的比例抽象出规律，>0.5概率即为幸存，<0.5即不能幸存，作为预测模型；
  • 其他步骤与简单实现步骤相同。

利用Python实现数据分析的基本流程

  1. 导入train.csv文件，转换为数组。需要用到csv和numpy函数；
  2. 用脚本语言计算性别幸存比例；
  3. 打开test.csv，按照预测模型，把幸存者预测写到一个新文件中，提交。
  • 如果想要复杂一点，在第2步中可计算幸存与票价、等级和性别三者的联系，将这些类别幸存概率值保存为一个2*3*4的矩阵。通过矩阵值观察概率，得出预测模型；其他按照上面步骤进行。

复杂之处在于计算train.csv概率矩阵时需要用到嵌套循环；在使用预测模型推断test.csv的幸存值时，也需要用到嵌套循环。

所用到的基础知识：如何打开关闭文件，如何将文件转化为数组，如何抽取某列，如何遍历所有乘客等。

用Python实现时掉进的三个坑：

  • 打开文件时的引用路径：

    csv_file_object = csv.reader(open('Downloads/train.csv', 'rb'))

以上需要用到绝对路径，如何使用相对路径我还没找到方法。现在使用的工具是iPython + notebook，对一些背景知识不熟悉。

  • 不能使用／#，而要用#

    women_only_stats = data[                          \#Which element

在调试上面部分时总是报错，尝试了几次发现删除／就OK了。教程上是“\#” 。

    for i in xrange(number_of_classes):       #loop through each class
  for j in xrange(number_of_price_brackets):       #loop through each price bin
